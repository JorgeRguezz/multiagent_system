3.4 query_analyzer.py
Implement lightweight intent detection.

Function:
- analyze_query(query: str) -> QueryIntent

Minimum behavior:
- normalize whitespace
- lowercased helper copy for heuristics
- set is_cross_video True if query includes terms like:
  - "across videos", "compare", "which video", "most videos"
- set is_temporal True if query includes:
  - "when", "timeline", "before", "after", "at what time"
- set is_visual_detail True if query includes:
  - "what appears", "what is shown", "in the frame", "visual"
- entity_focus_terms:
  - simple noun-like term extraction + champion-like terms from query tokens

Note: This is intentionally simple first pass.

3.5 retrievers.py
Implement parallel hybrid retrieval.

Functions:
- retrieve_chunks_dense(query: str, stores: dict[str, VideoStore], k: int) -> list[RetrievalHit]
  - Query each store.chunks_vdb with text query
  - Resolve chunk text + segment ids from chunks_kv

- retrieve_entity_graph(query: str, stores: dict[str, VideoStore], k_entities: int, k_graph_chunks: int) -> list[RetrievalHit]
  - Query each store.entities_vdb
  - For matched entities, find node in graph
  - Collect source_id chunk IDs from nodes + 1-hop neighbors
  - Convert to RetrievalHit with source="entity_graph"

- retrieve_global_graph(query: str, global_graph: nx.Graph, stores: dict[str, VideoStore], k: int) -> list[RetrievalHit]
  - Match candidate nodes in global graph by simple string similarity / contains
  - Collect chunk IDs from source_id
  - Map chunk IDs back to owning video store by membership in chunks_kv

- retrieve_visual_support(...) -> list[RetrievalHit]
  - OPTIONAL first version:
    - use frame text only when intent.is_visual_detail is True
    - do not run expensive image model in v1

- retrieve_all(query, intent, stores, global_graph) -> list[RetrievalHit]
  - Run retrieval branches in parallel with asyncio.gather
  - Merge and return flat hits

Important:
- Always include source label: dense_chunk / entity_graph / global_graph / visual_support.
- Do not silently drop exceptions; log and continue per branch.

3.6 reranker.py
Implement dedupe + weighted ranking + diversity.

Functions:
- dedupe_hits(hits) -> list[RetrievalHit]
  - key = (video_name, chunk_id)
  - keep best component scores

- compute_component_scores(hits, query, intent) -> list[RetrievalHit]
  - semantic score already from VDB
  - entity score based on overlap between query terms and chunk text
  - graph score based on graph-derived source and neighbor support

- apply_weighted_score(hits) -> list[RetrievalHit]
  - final_score = weighted sum from config

- apply_diversity(hits, max_per_video: int) -> list[RetrievalHit]
  - cap evidence concentration per video unless query is clearly single-video

- rerank_hits(...) -> list[RetrievalHit]
  - orchestrates all above and returns sorted top-N

3.7 context_builder.py
Convert ranked hits to prompt context.

Functions:
- resolve_time_span(store: VideoStore, segment_ids: list[str]) -> str
  - parse segment time strings and return concise range
- make_evidence_blocks(hits, stores, budget_tokens) -> list[EvidenceBlock]
  - build blocks with provenance
  - enforce context budget and stop when full
- render_context_for_prompt(evidence_blocks) -> str
  - format each block as:
    [video=<name> | time=<h:mm:ss-h:mm:ss> | chunk=<id> | source=<source>]
    <content>

Budgeting:
- Truncate long chunks with explicit marker "[truncated]".
- Prefer many medium blocks over few huge blocks.

--------------------------------------------------------------------------------
7) OPEN QUESTIONS / UNDEFINED PARTS (DO NOT HIDE)
--------------------------------------------------------------------------------

These items are intentionally not fully fixed and require decision/tuning:

1. Query intent method:
- heuristic-only vs cheap-LLM-assisted intent detection.
- Tradeoff: speed vs nuanced routing.

2. Exact scoring weights:
- Initial weights are placeholders; tune with eval set.

3. Diversity policy:
- strict max chunks per video vs adaptive based on dominance confidence.

4. Verifier strictness:
- aggressive pruning of unsupported claims can over-shorten answers.
- Alternative: keep answer but label uncertain sections.

5. Cross-video global graph retrieval algorithm:
- simple lexical node matching vs embedding-based node retrieval.
- Lexical is faster and simpler for v1.

6. Visual retrieval depth in v1:
- text-only frame evidence vs multimodal embedding retrieval.
- Recommend text-only first for speed/stability.

7. Confidence calibration:
- Current formula is heuristic; should be calibrated on validation QA set.

8. Maximum context size:
- Needs tuning against quality/latency constraints and model behavior.

9. How to handle conflicting evidence across videos:
- Option A: majority evidence
- Option B: report both interpretations with provenance

10. Evaluation ground truth quality:
- If expected answers are weak/noisy, tuning may optimize wrong behaviors.
